## 😰 私が陥った勘違い

**これは幻想でした。**

実際にやってみたら…

|---|---|---|---|
| スループット | 50-60 tok/s | **0.9 tok/s** | 😱 50倍遅い |
| GPU 使用率 | 80%+ | **24%** | 😱 GPU 寝てる |

## 🔬 何が起きていたか：CPU offload 地獄

GPU 使用率 24% / 消費電力 106W が示すのは、**GPU が CPU を待っている時間が大半**だということ。VRAM に入りきらないモデルレイヤーが CPU に退避し、推論のたびに **PCIe 経由で重みが行き来**していたのです。

### VRAM の中身を分解

| 内訳 | 容量 |
|---|---|
| モデル本体 (Q8_0) | 29 GB |
| KV cache (32K context) | 約 3-4 GB |
| 推論オーバーヘッド | 約 0.5 GB |
| **合計 必要 VRAM** | **約 33 GB** |
| **RTX 5090 VRAM** | **32 GB** |
| **不足分** | **約 1 GB** |

**たった 1GB のオーバーで全レイヤーが減速**します。GPU/CPU 間のメモリ転送速度は GDDR7 の 1/30 以下。レイヤー単位で行き来すれば、それだけで桁違いに遅くなります。

## 💡 試した対策と結果

### 対策 1: FlashAttention 有効化

VRAM 使用量を削減する標準的な手法。

```powershell
[System.Environment]::SetEnvironmentVariable("OLLAMA_FLASH_ATTENTION", "1", "User")
[System.Environment]::SetEnvironmentVariable("OLLAMA_KV_CACHE_TYPE", "q8_0", "User")
```

**結果: わずか 11% 改善**（217s → 193s）。VRAM ピークも 32.4GB → 32.4GB と変わらず。**根本解決にはならない**。

> 詳細は **【FlashAttention 編】**で別途検証します。

### 対策 2: コンテキスト長を短くする

```python
options = {"num_ctx": 4096}  # 32768 → 4096
```

KV cache が 1/8 になるので 2-3GB 節約できる可能性あり。ただし長いコード解析ができなくなる副作用。

### 対策 3: Q6_K に下げる

bartowski から Q6_K (23GB) を取得すれば、9GB 余裕ができる。**現状ではこれが最も現実的**。

### 対策 4: マルチ GPU

RTX 5090 を 2 枚（合計 64GB VRAM）にすれば Q8_0 が完全に乗ります。ただし投資 100 万円コース。

## 🎓 学んだこと：「VRAM サイズ ≠ 動かせるモデルサイズ」

ローカル LLM を選ぶときは、以下の式で考えるべき:

```
動かせるモデルサイズ = VRAM サイズ × 0.85 - KV cache 概算

例: RTX 5090 (32GB)
  → 32 × 0.85 - 4 = 23.2 GB が安全圏
  → Q6_K (23GB) ぴったり、Q8_0 はアウト
```

「サイズ ≤ VRAM」ではなく、**「サイズ ≤ VRAM × 0.85 - KV cache」** で見るのがコツです。

## ⚠️ ハマりポイント

### 1. ベンチマーク 1 回目だけ遅い

Ollama はモデル初回ロードに 30-60 秒かかります。これを TTFT に含めて測ると数値が悪く見えるので、**ウォームアップ後に計測**するのが正確。

### 2. Windows と Linux で挙動が違う

Windows の Ollama は CPU offload が積極的。Linux/WSL2 では同じ Q8_0 でも違う挙動の報告あり。

### 3. nvidia-smi の VRAM が「瞬間値」

`32431 MiB` と出ても、瞬間的に 32500 を超えていれば、その時点で OOM 回避のため CPU 退避が始まっています。**1秒間隔のサンプリング**では検出できないことも。

---

## 📝 まとめ：Q8_0 を試す前に確認すべき 3 ステップ

1. **ハードの VRAM 容量 × 0.85 を計算** → モデルサイズより小さければ NG
2. **必要な KV cache サイズを試算**（context長 × hidden_size × layers）
3. **少しでもギリギリなら 1 段下のモデルを使う**（Q8 → Q6_K）
