## 💰 損益分岐点：Claude Code Max vs ローカル LLM

### 月額の直接費用

| 項目 | Claude Code Max | ローカル (RTX 5090) |
|---|---|---|
| 月額固定費 | **15,000 円** | 0 円 |
| 電気代 (8h × 20日) | 0 円 | **約 1,500 円** |
| **直接コスト** | **15,000 円** | **1,500 円** |
| **月差** | — | **13,500 円安** |

### 「見えないコスト」の試算

これだけだと「年 16 万円の節約か、まあアリ」程度ですが、Claude Code Max の **隠れた損失**を入れると景色が変わります。

| 損失項目 | 月の影響（私の場合）|
|---|---|
| 5h 制限で作業中断 → 別作業への切替コスト | 月 5-8 時間 |
| 性能劣化時間帯のリトライ・諦め | 月 10-20 時間 |
| 機密案件で使えない → 自力作業 | 月 5-10 時間 |
| **合計時間損失** | **月 20-38 時間** |

時給 5,000 円換算で **月 10-19 万円相当の損失**。これが解消されるとなると、回収計算は別物になります。

### 投資回収期間の比較

```text
RTX 5090 価格: 約 50 万円

【直接費用ベース】
  月差: 13,500 円
  回収期間: 50万 ÷ 13,500 = 約 37 ヶ月（3 年）

【時間損失込みベース】
  月差: 13,500 + 100,000 = 113,500 円相当
  回収期間: 50万 ÷ 113,500 = 約 4-5 ヶ月

【最大ケース】
  月差: 13,500 + 190,000 = 203,500 円相当
  回収期間: 約 2-3 ヶ月
```

「制限・劣化に頻繁に困っているヘビーユーザー」ほど、**ハードウェア投資が早期に回収**できる構造です。

## 🧠 5 記事を経て得た「3 つの教訓」

### 教訓 1: 量子化を上げても品質は劇的に上がらない

Q4 と Q8 の品質差は **6 タスク正答率で同じ（6/6）**。応答の語彙バリエーションは Q8 の方がやや豊かだが、**コーディング作業の質には影響しない**。

### 教訓 2: VRAM はモデルサイズの 1.2 倍以上必要

```text
安全圏: モデルサイズ × 1.2 < VRAM
  - Q4 (17GB) × 1.2 = 20GB < 32GB ✓
  - Q5 (20GB) × 1.2 = 24GB < 32GB ✓
  - Q8 (30GB) × 1.2 = 36GB > 32GB ✗ ← 32GB VRAM では NG
```

KV cache とオーバーヘッドで実は **モデルサイズの 1.2 倍程度の VRAM が要る**。

### 教訓 3: ベンチマークしてから設定を変える

「FlashAttention は速くなる」「num_ctx 大きい方がいい」など、**鵜呑みにせず必ず自環境で計測**。今回も Q4/Q5 で逆効果という意外な結果が出た。
